## ğŸ“ Recording Your First Meeting

### When to Record a Meeting
- Student has missed multiple classes
- You've met with the student to discuss absences
- You need to document the conversation and create an action plan

### Steps

1. **Click "Record New Meeting"** button (top right)

2. **Select Student** (if not pre-selected)
   - Choose from dropdown list

3. **Enter Meeting Date**
   - Date when you met with the student

4. **Document Absence Reason**
   - Write exactly what the student told you
   - Example: "Student said they were sick with flu for 3 days"

5. **Select Root Cause Category**
   - Health Issue
   - Family Issue
   - Academic Difficulty
   - Financial Issue
   - Behavioral Issue
   - Other

6. **Write Intervention Notes**
   - Detailed observations from the meeting
   - Student's attitude and responses
   - Any concerns or red flags
   - Example: "Student seemed genuine. Parent confirmed illness via phone."

7. **Define Action Plan**
   - Specific steps you'll take
   - What you expect from the student
   - Example: "Monitor attendance for next 2 weeks. Student to provide medical note if absent again."

8. **Set Follow-up Date** (optional but recommended)
   - When you'll check in again
   - Must be a future date

9. **Choose Urgency Level**
   - **Low**: Minor issue, routine follow-up
   - **Medium**: Needs attention, monitor closely
   - **High**: Serious concern, immediate action required

10. **Click "Record Meeting"**

## ğŸ“Š Tracking Progress

### When to Add Progress Updates
- After each follow-up conversation
- When you observe changes in behavior
- When status changes (improving/not improving)
- Before closing the case

### Steps

1. **Find the meeting** in the table

2. **Click "Track Progress â†’"**

3. **Review meeting summary** at the top

4. **Update Status** if needed:
   - **Open**: Just created, no action yet
   - **Monitoring**: Actively watching the situation
   - **Improving**: Student showing positive changes
   - **Not Improving**: No progress or getting worse
   - **Escalated**: Sent to administration
   - **Closed**: Issue resolved

5. **Write Progress Notes**
   - What happened since last update
   - Current situation
   - Any new information
   - Example: "Student attended all classes this week. Spoke with parent - situation at home has stabilized."

6. **Click "Add Progress Update"**

## âœ… Best Practices

### 1. Document Immediately
- Record meetings the same day
- Details are fresh in your mind
- More accurate documentation

### 2. Be Specific
- Avoid vague statements
- Include concrete examples
- Quote the student when relevant

### 3. Set Follow-ups
- Always set a follow-up date
- Creates accountability
- Ensures nothing falls through cracks

### 4. Update Regularly
- Add progress notes weekly
- Keep status current
- Build a complete history

### 5. Close Cases Properly
- Add final summary
- Document outcome
- Change status to "Closed"

### 6. Escalate When Needed
- Don't wait too long
- If not improving after 3-4 updates, escalate
- Better to escalate early than too late
